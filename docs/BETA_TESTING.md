# 베타 테스트 가이드

## 개요

Fashion Influencer Matcher 베타 테스트를 통해 실제 사용자 피드백을 수집하고 제품을 개선합니다.

---

## 테스트 목표

1. **기능 검증**: 핵심 기능이 예상대로 작동하는지 확인
2. **성능 측정**: 분석 완료 시간, API 오류율 측정
3. **UX 피드백**: 사용자 경험 개선점 파악
4. **점수 정확도**: 브랜드-인플루언서 매칭 점수의 실제 적중률 확인

---

## 테스트 시나리오

### 시나리오 1: 기본 분석 흐름

```
목표: 브랜드와 인플루언서 분석 및 결과 확인

단계:
1. 홈페이지에서 "분석 시작하기" 클릭
2. 브랜드 Instagram username 입력 (예: @musinsa)
3. 인플루언서 3-5명 username 입력
4. "분석 시작" 클릭
5. 진행 상태 확인 (queued → running → done)
6. 결과 대시보드 확인
7. 상세 분석 페이지 클릭

예상 결과: 
- 분석 완료 시간: 2-5분
- 모든 인플루언서에 점수 및 등급 표시
- CSV 납품내기 가능
```

### 시나리오 2: 계정 검증 및 에러 처리

```
목표: 잘못된 계정 입력 시 적절한 에러 메시지 표시

단계:
1. 존재하지 않는 username 입력 (예: @notexist12345)
2. 개인 계정 username 입력
3. 비공개 계정 username 입력
4. 특수문자가 포함된 username 입력

예상 결과:
- 명확한 에러 메시지 표시
- 해결 방법 안내
- 대체 입력 유도
```

### 시나리오 3: 캐시 및 재분석

```
목표: 캐시된 데이터 확인 및 재분석 흐름

단계:
1. 특정 브랜드 분석 (첫 요청)
2. 동일 브랜드 재분석 요청 (캐시된 데이터 사용)
3. 6시간 후 동일 브랜드 재분석 요청 (새로운 데이터)

예상 결과:
- 첫 요청: API 호출 발생 (2-3초 소요)
- 캐시 히트: 즉시 결과 반환 (<500ms)
- 6시간 후: 새로운 API 호출
```

### 시나리오 4: 대량 분석 및 Rate Limit

```
목표: Rate limit 상황에서의 동작 확인

단계:
1. 연속으로 10개 이상의 분석 작업 요청
2. 200콜/시간 한도 도달
3. 대기 상태에서의 UI 피드백 확인

예상 결과:
- 큐에 작업이 순차적으로 적재
- Rate limit 임박 시 경고 표시
- 예상 대기 시간 표시
- 자동 재시도
```

### 시나리오 5: 데이터 삭제

```
목표: 데이터 삭제 기능 확인

단계:
1. 분석 작업 완료
2. 설정 페이지에서 "내 데이터 삭제" 클릭
3. 삭제 확인
4. 삭제 후 동일 분석 재요청

예상 결과:
- 삭제 완료 메시지 수신
- 삭제된 데이터 복구 불가
- 새로운 분석 시 새로운 API 호출
```

## 피드백 수집 방법

### 1. 인앱 피드백

```
위치: 대시보드 상단 또는 설정 페이지
형식: 5점 척도 + 텍스트 입력
질문:
- 분석 결과가 유용했나요? (1-5)
- 어느 부분이 가장 도움이 되었나요?
- 개선하고 싶은 점이 있나요?
- 다시 사용하실 의향이 있나요?
```

### 2. 정기 인터뷰

```
주기: 주 1회 (30분)
대상: 활발히 사용하는 베타 테스터 3-5명
형식: 화상 미팅 또는 전화
주요 질문:
- 이번 주에 어떤 기능을 사용했나요?
- 가장 만족스러웠던 점은?
- 가장 불편했던 점은?
- 새로 추가되었으면 하는 기능은?
- 가격(월 ₩29,000)에 대해 어떻게 생각하나요?
```

### 3. 사용 데이터 분석

```수집 항목:
- 분석 완료율 (성공 작업 / 전체 작업)
- 평균 분석 완료 시간
- API 오류율
- 재사용률 (주 2회 이상 사용 사용자)
- CSV 납품내기 사용 빈도
```

## 성공 기준 (Exit Criteria)

베타 테스트를 성공적으로 마치기 위한 기준:

### 기능
- [ ] 분석 완료율 95% 이상
- [ ] 평균 분석 시간 5분 이내
- [ ] API 오류율 2% 이하

### 사용자
- [ ] 재사용률 50% 이상 (주 2회+)
- [ ] NPS 50점 이상
- [ ] 심각한 버그 리포트 0건

### 비즈니스
- [ ] 10개 유료 전환 (Pro 플랜)
- [ ] 3개 이상의 긍정적 케이스 스터디
- [ ] 킬 크라이테리아 미충족 (6/10 WTP)
